# Product Roadmap

## Document status
- Owner: Product + Engineering Leadership
- Last updated: 2026-02-11
- Horizon: Next 4 quarters

## Roadmap view

### Q1: Foundation hardening
- Unify documentation into complete product set (done in this update).
- Identify SQL/ORM contract mismatches and publish convergence plan.
- Improve baseline QA checks and release-note discipline.

### Q2: Reliability and trust
- Harden auth flows and security controls.
- Expand moderation traceability and review SLA tracking.
- Add fallback and observability improvements for external API dependencies.

### Q3: Personalization and growth
- Improve recommendation relevance from user behavior signals.
- Optimize subscription conversion funnel.
- Enhance affiliate and partner reporting surfaces.

### Q4: Scale and platform maturity
- Formalize API contracts for integration use.
- Improve automation for deployments, checks, and compliance workflows.
- Evaluate modularization options for key business domains.

## Dependencies
- External API stability (OpenAI, Stripe, RSS sources).
- Team bandwidth for testing and refactoring.
- Legal policy alignment as monetization expands.

## Change rationale
A staged roadmap balances immediate reliability needs with growth features while preserving maintainability.
